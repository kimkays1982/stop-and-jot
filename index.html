Stop & Jot — Screen-by-Screen Design Rationale

What Marisol sees, does, and clicks. Design decisions at every step. · Kim Kays · June 2026

Design language

The visual grammar behind every choice. Color carries semantic meaning. Type scale is calibrated for Chromebook at a 13" screen with potential for low-vision users. Spacing defaults to touch-friendly minimums.

Type scale
Article body / primary UI
13px — system sans-serif
Matches the real Medley extension body. Dense but legible at 13"; nothing smaller for reading text.
Tab labels / mode buttons
12px — medium weight
Slightly smaller to distinguish navigation from content; still meets 11px minimum for school Chromebooks.
Meta labels / tag chips
10PX MEDIUM
Tags and notices. Letter-spacing added at this size to preserve readability — tight spacing at 10px collapses letterforms.
Section eyebrows / column headers
9PX CAPS — CATEGORY LABEL
All-caps at 9px only for true labels that categorize, never for readable sentences. Used sparingly.
Font family
System sans-serif stack
-apple-system, BlinkMacSystemFont, Segoe UI. No web font load = no FOUT, no latency penalty. School devices often have slow connections.
Design principles
Low cognitive load by default
No animation. No modal overlays. No notification banners. Every visual element that appears was put there; nothing appears on its own. The student's attention belongs to the text.
Agency before scaffolding
The student activates Stop & Jot. The student chooses their response mode. The student can skip. Scaffolds that activate automatically become invisible — students stop noticing them and stop deciding. Agency is the feature.
Separation of concerns
Each tab in the extension does one thing. Amplifier modifies text. Stop & Jot captures thinking. Shorten reduces text. Translate switches language. No feature bleeds into another's job. This prevents interface clutter and makes the mental model learnable.
SpEd + ELL is the design case, not a variant
The primary user has an IEP, is a multilingual learner, or both. All defaults, all copy, all interaction states are designed for that student first. Students with fewer needs use fewer affordances. No one gets routed to a different experience.
Accessibility decisions
✓ Touch targets ≥ 36px — all interactive elements
✓ #3B6FD4 on white: 4.6:1 contrast ratio (AA)
✓ No animation by default — reduces vestibular triggers
✓ Color never the sole signal — state changes use shape + text, not color alone
✓ Skip is always visible — no hidden escape from cognitive demand
✓ System font stack — no web font dependency on slow school networks

1

Stop & Jot is its own tab, not embedded in Amplify

Amplify modifies the text. Stop & Jot needs the original. Bundling them into one tab lets students amplify and jot at the same time, which is over-scaffolding. A dedicated tab makes them two separate modes the student chooses between. That's by design.

2

Student has to click in, nothing auto-launches

The extension doesn't open Stop & Jot when a student navigates to an article. They activate it. Agency starts before the first prompt. Medley's whole design philosophy is that scaffolds are tools you reach for, not things that happen to you.

3

Extension goes quiet once reading starts

No banner, no countdown, no visual noise. The card is the only signal the student gets. The goal is to keep cognitive load on the reading itself and only interrupt at the breakpoint.

1

Card is inline, not a modal or separate page

The card surfaces at exactly the location in the text where the breakpoint falls. A modal removes the student from context. A separate page breaks reading flow. Inline lets Marisol scroll back and re-read while she's responding. That's the point of a metacognitive pause.

2

First question is always literal recall

Opening with inference is a barrier for a student with SLD and emerging English. Literal recall is the entry ramp. Questions build from there within the same reading: prediction, connection, synthesis. The progression is constrained in the LLM prompt, not left to chance.

3

Default response mode is driven by the student profile

Marisol's Lexile and WIDA profile set the default tab automatically. No teacher configures this. A student with stronger written expression opens to the Type tab instead. The card adapts to the student, not the other way around.

4

All three modes and Skip are visible, nothing is buried

Type, Voice, Sentence starter, and Skip are all present. Skip is a first-class option, not an escape hatch. Forcing a response before the student can continue creates punitive friction, especially for students with IEPs around task initiation. A student who skips is still reading.

5

Continuation text dims, reads as preview not wall

The text below the card is lighter but still scrollable. It signals "the article continues" without removing context. No animation, no overlay. The card is a static embed inline in the text. Animation adds cognitive load and for SpEd students, static is the right default.

1

Starter pre-fills the input, student adds their own words

Tapping a starter fills the text field with the frame but leaves a blank to complete. Marisol finishes it in her own words. The scaffold is a launch pad, not the answer. The student still has to produce something, not just select.

2

System logs mode and language choice for calibration

Marisol chose Sentence starter in English. The system stores this. Future jots in the same session calibrate scaffold density from usage. If she keeps using starters, density stays high. If she's typing freely, starters fade back. The scaffold adapts rather than locking her into a preset level.

3

Submit replaces Skip once text is in the field

Once Marisol has text in the field, Skip swaps to Submit. Skip and Submit should never compete for the same slot. A student who's written something and clicks Skip would lose the response. That's a trust-breaking moment, especially for SpEd students with effort fatigue.

1

The full card localizes, not just the question

"En español" switches everything: starters, mode labels, feedback, confirmation text. The article itself stays in English. That's the Translate tab's job. Keeping them separate avoids collapsing two different modes into one.

2

Feedback is metacognitive, never a grade or score

The Reflexión panel acknowledges what Marisol noticed and points forward to the next section. No score, no "great job," no correct or incorrect. The LLM is constrained to three response types: reflect-and-extend, gentle redirect, neutral acknowledgment. Evaluative feedback turns the jot into a quiz. That defeats the whole strategy.

3

"Añadido a tu diario" is a deliberate copy choice

"Saved," "submitted," or "recorded" all signal institutional ownership. "Added to your journal" puts it back in the student's hands. That framing shapes how students think about the feature over time: study tool or compliance task.

4

SpEd and ELL is the primary design case, not an edge case

Many students with IEPs are also multilingual learners. The architecture is built around that profile, not the average student. Same extension, same card, same tab. Students with fewer needs use fewer affordances. No separate experience, no routing.

1

Session-only persistence, named explicitly in the UI

Jots persist for the session and clear when it closes. The notice tells the student directly: "Saved for this reading. Copy to keep them." Cross-session persistence needs a student portal or LMS integration. Both are V2. This is a deliberate scope call, not a gap we missed.

2

Copy and paste is the right V1 export, not a compromise

Copy needs no LMS integration, no new data pipeline, no engineering lift beyond the button. It delivers something real: a portable record of thinking that can go into Google Classroom, a parent email, or a personal note. V1 ships what works. V2 adds direct LMS export to Canvas or Schoology. The architecture doesn't close those doors.

3

Same student, different choices per jot

Jot 1: Spanish, sentence frame. Jot 2: English, voice. The tags show this at a glance. The student was never locked into a single mode. The architecture supports mixing within one session, which is exactly what a SpEd student whose needs shift day to day actually needs.

4

Question progression is visible in the trail, literal to synthesis

Jot 1 was literal recall. Jot 2 required synthesis. The trail shows Marisol how her thinking developed across the reading. That's what makes this a study tool, not a compliance log. The progression isn't random. The LLM is constrained to a defined question type sequence.

1

Three new tiles and one new column, no new dashboard

Everything plugs into the dashboard that already exists. No new infrastructure, no new teacher habit to build. Stop & Jot extends what's there. We're adding to existing data patterns, not creating new ones.

2

Teachers see engagement signals, not response content

Completion rates, scaffold usage, and a low-engagement flag for students under 50%. That's it. No response text visible by default. When students think every response gets reviewed, they write for approval instead of writing to think. Teachers who want the responses ask students to copy them into a Google Classroom assignment.

3

Low-engagement flag is a pattern signal, not surveillance

James T. at 25% surfaces as a flag on the dashboard. Not a notification, not a real-time alert. Skip patterns aggregate over time, they don't flag individual moments. The signal is useful for a teacher check-in, not a surveillance trigger. Real-time push alerts are V2 and worth a real conversation before building.

4

Teacher content visibility is a V2 decision, named not buried

The tension is real. Teachers want comprehension insight. Students need safety to respond honestly. V1 doesn't resolve it, and that's intentional. The V2 hypothesis is AI-summarized misconception reports rather than raw responses. Teachers get instructional signal without individual exposure. That's worth designing carefully, not shipping fast.

Patterns we ruled out — and why

Design judgment is as much about what you don't build. Each of these patterns is common in ed-tech. Each one had a reason it didn't make it in.

✕
Modal / overlay card
A modal removes the student from the text they just read. The jot question needs to coexist with the passage — not replace it. Inline is the feature. A modal is a different product.
✕
Auto-launch on page load
If the card appears automatically, students learn to dismiss it before reading. Activation has to be intentional or the scaffold becomes noise. Agency starts before the first prompt — not after you've already closed a dialog.
✕
Score or grade on the jot feedback
A score turns a metacognitive tool into a quiz. Students write for approval instead of writing to think. The LLM prompt is constrained to reflect, redirect, or acknowledge — never evaluate. Correct/incorrect is not a feedback mode here.
✕
Animated card entrance
Slide-in and fade-in animations add cognitive load and can trigger vestibular symptoms in students with certain IEPs. Static is the right default. Motion can be a V2 opt-in for users who prefer it — not the other way around.
✕
Teacher-configures-scaffold-level
If the teacher sets the scaffold level, every student in the class gets the same default. That's the old model. The system reads WIDA and Lexile from the existing profile and sets defaults automatically. Teachers confirm what's working; they don't spec the starting point.
✕
Collapsing the card after submission
If the card collapses immediately after submit, the student loses the question context before they've fully processed the feedback. The feedback state stays visible until the student scrolls past it. The student closes it by reading forward, not by a dismiss button.
✕
Real-time teacher notification on skip
Push alerts for every skip would make skipping feel like a behavior report. Skip is a first-class option — students shouldn't feel monitored for using it. Engagement signals aggregate over time and surface in Usage Reports as patterns, not individual events.
✕
Jot card embedded inside Amplifier
Amplify modifies the text. Stop & Jot needs the original. Combining them in one tab means students would amplify and jot simultaneously — that's over-scaffolding. Two separate modes requires two separate surfaces. The tab architecture enforces that cleanly.
